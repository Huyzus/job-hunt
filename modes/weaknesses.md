# Mode: weaknesses — Pattern Analysis from Debriefs

## Purpose

Surface recurring weakness patterns that are invisible from inside any single interview. Reads `data/weakness-map.md` (auto-generated by `analyze-interview-patterns.mjs`) and renders a prescription card by default, with optional appendix views for full evidence.

## When to Invoke

- User runs `/job-hunt weaknesses` (on-demand)
- Auto-fired by `debrief` mode Step 9.5 when `threshold_hit: true`

## Inputs

1. **No required arguments**
2. **Optional flags:**
   - `--full` → render the appendix (per-stage breakdown, evidence quotes, cross-pattern observations)
   - `--evidence N` → expand evidence for pattern #N from the Top Patterns table

## Step 1 — Run the Analyzer

Invoke: `node analyze-interview-patterns.mjs --json`

Capture stdout JSON:
```json
{
  "patterns_total": 8,
  "high_confidence": 2,
  "medium_confidence": 3,
  "threshold_hit": true,
  "fired_reason": "...",
  "since_last_run_days": 4,
  "file_changed": true
}
```

If invocation fails or returns no JSON, surface the error and stop.

## Step 2 — Read the Weakness Map

Read `data/weakness-map.md`. The script just wrote it (or confirmed it's unchanged).

If the file says "Not yet seeded", print:
> "Not enough data yet. Run more `/job-hunt debrief` sessions to begin pattern tracking."

Stop here.

## Step 3 — Render the Prescription Card (default view)

Extract from weakness-map.md:
- `## Top Patterns (sorted by confidence)` table → first 3 rows
- `## Win Patterns (what works — leverage these)` table → first row

Render to user:

```
🎯 Weakness Map — {N} debriefs analyzed (last 90 days)

TOP 3 PATTERNS:
1. {confidence_emoji} {Confidence} — {Pattern} ({Frequency})
   → Drill: {Drill text from table}
2. {confidence_emoji} {Confidence} — {Pattern} ({Frequency})
   → Drill: {Drill text}
3. {confidence_emoji} {Confidence} — {Pattern} ({Frequency})
   → Drill: {Drill text}

WIN PATTERN TO LEVERAGE:
✅ {Win pattern} — {Evidence}

→ See full analysis: /job-hunt weaknesses --full
→ See evidence for #1: /job-hunt weaknesses --evidence 1
→ Suppress auto-trigger: edit config/profile.yml → weaknesses.auto_trigger: false
```

`{confidence_emoji}`: `⚠️` for High, `📊` for Medium, `📋` for Low. (If fewer than 3 patterns exist, render only the rows that have data.)

## Step 4 — Render the Appendix (only if --full or --evidence N)

Read source files in addition to weakness-map.md to add LLM-driven synthesis:
- `cv.md` — to ground recommendations in user's actual proof points
- `interview-prep/story-bank.md` — to tailor drills to existing stories
- 2-3 most recent debriefs from `interview-prep/debriefs/` — for verbatim evidence quotes

For `--full`: render every section of weakness-map.md, plus:
- **Cross-pattern observations**: synthesize whether two patterns may really be one (e.g., "VP-Design losses overlap with business-framing fumbles — these may be one pattern, not two")
- **Tailored drill recommendations**: instead of generic templates, generate drills that reference specific stories from story-bank
- **Evidence per pattern**: for each Top Pattern, show 2-3 verbatim quotes from the debrief files that contributed

For `--evidence N`: render only the evidence section for the Nth pattern in the Top Patterns table.

## Step 5 — Auto-Trigger Context

If invoked from `debrief` Step 9.5 (the caller passes `--auto` or sets context), prepend the card output with:

> "🚨 Pattern threshold crossed: {fired_reason}. Surfacing while details are fresh."

## Edge Cases

- **<3 debriefs total**: print "Not enough data yet ({N}/3 debriefs). Run more debriefs to see patterns." Skip card.
- **All confidence Low**: print "No high-confidence patterns yet. {N} debriefs analyzed but signals are too sparse."
- **weakness-map.md missing**: print "Weakness map missing. Run `/job-hunt debrief` to seed it." Stop.
- **JSON parse failure**: print "Analyzer returned invalid output. Try `node analyze-interview-patterns.mjs --dry` to debug."

## What This Mode Does NOT Do

- Does not analyze (the `.mjs` script's job)
- Does not write `data/weakness-map.md` (also the script's job)
- Just orchestrates: run analyzer → render output → handle args
